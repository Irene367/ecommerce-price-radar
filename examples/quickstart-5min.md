# 5 分钟上手

## 1. 最快开始

复制 `templates/minimal-daily-input.md`，填入本店商品、到手价、最低毛利要求，以及 1-3 个重点竞品的价格、优惠、库存和发货承诺。

然后直接问：

```markdown
请判断今天要不要跟价。如果不建议直接跟价，请给出非价格防守策略和今日行动清单。
```

## 2. 最小可用输入模板

```markdown
本店商品：
本店到手价：
单件总成本：
最低毛利要求：

竞品数据：
| 竞品 | 竞品分层 | 平台 | 昨日到手价 | 今日到手价 | 优惠/活动 | 库存 | 发货承诺 | 评价/卖点变化 |
| --- | --- | --- | ---: | ---: | --- | --- | --- | --- |
|  |  |  |  |  |  |  |  |  |
```

## 3. 极简示例输入

```markdown
本店商品：便携榨汁杯 A 款
本店到手价：79 元
单件总成本：48 元
最低毛利要求：25%

竞品数据：
| 竞品 | 竞品分层 | 平台 | 昨日到手价 | 今日到手价 | 优惠/活动 | 库存 | 发货承诺 | 评价/卖点变化 |
| --- | --- | --- | ---: | ---: | --- | --- | --- | --- |
| 竞品1 | 直接竞品 | 淘宝 | 82 | 72 | 新增10元限时券 | 充足 | 48小时内发货 | 评价稳定，强调防漏 |
| 竞品2 | 低价扰动竞品 | 拼多多 | 76 | 75 | 6元券 | 充足 | 未说明 | 有差评提到噪音 |
```

## 4. 预期输出结构

输出会包含：

1. 本次识别模式
2. 决策卡片
3. 结论摘要
4. 数据来源与置信度
5. 关键变化
6. 风险等级
7. 调价建议
8. 非价格防守策略
9. 缺失数据与影响
10. 今日行动清单
11. 需要用户确认的问题
